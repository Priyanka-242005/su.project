Servlets and JSP

Understanding Servlets

A servlet is a Java class that handles HTTP requests and responses. It runs on a web server or application server and is used to create dynamic web content.

Servlet Lifecycle

Servlets follow a specific lifecycle managed by the servlet container:

  • Loading: The servlet class is loaded into memory.
  • Instantiation: An instance of the servlet is created.
  • Initialization: The init() method is called.
  • Request Handling: The service() method is called for each request.
  • Destruction: The destroy() method is called before the servlet is removed from memory.

Creating a Simple Servlet

Example:
import java.io.*;
import javax.servlet.*;
import javax.servlet.http.*;
public class HelloWorldServlet extends HttpServlet {
protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
PrintWriter out = response.getWriter();
out.println("Hello, World!");
}
}

Understanding JSP (JavaServer Pages)

JSP is a technology that helps in creating dynamically generated web pages based on HTML, XML, or other document types. It allows embedding Java code directly into the HTML markup.

JSP Lifecycle

The lifecycle of a JSP page includes:

  • Translation: The JSP is converted into a servlet by the server.
  • Compilation: The generated servlet is compiled.
  • Execution: The servlet is executed to handle requests.

Creating a Simple JSP Page

Example:
<%@ page language="java" contentType="text/html; charset=UTF-8" pageEncoding="UTF-8"%>
<!DOCTYPE html>
<html>
<head><title>Hello World</title></head>
<body>
<h1>Hello, World!</h1>
</body>
</html>

Benefits of Servlets and JSP

  • Platform-independent and portable.
  • Separation of business logic from presentation layer.
  • Support for reusable components.
