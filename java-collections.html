Collections Framework

What is the Collections Framework?

The Java Collections Framework provides a set of classes and interfaces for storing and manipulating groups of data as a single unit. Collections in Java allow for dynamic storage, retrieval, manipulation, and access to data in an efficient manner.

Key Interfaces of the Collections Framework

  • Collection: The root interface of the Java Collections Framework. It provides basic methods like `add()`, `remove()`, `size()`, and `clear()` for working with collections of objects.
  • List: An ordered collection that allows duplicate elements. Commonly used implementations include `ArrayList`, `LinkedList`, and `Vector`.
  • Set: A collection that does not allow duplicates. Common implementations include `HashSet`, `LinkedHashSet`, and `TreeSet`.
  • Queue: A collection used for holding elements prior to processing, often in a first-in-first-out (FIFO) manner. Implementations include `LinkedList` and `PriorityQueue`.
  • Map: A collection that maps keys to values. No duplicate keys are allowed. Common implementations include `HashMap`, `TreeMap`, and `LinkedHashMap`.

Common Classes in the Collections Framework

  • ArrayList: A resizable array implementation of the `List` interface. It provides fast random access but slower insertion/removal in the middle of the list compared to `LinkedList`.
  • LinkedList: A doubly-linked list implementation of both `List` and `Deque` interfaces. It is ideal for frequent insertions and deletions.
  • HashSet: Implements the `Set` interface and is backed by a hash table. It doesn't guarantee any order of elements.
  • HashMap: Implements the `Map` interface and stores key-value pairs. It provides constant-time performance for the basic operations (get and put).
  • TreeMap: Implements the `Map` interface and provides a sorted map based on the natural ordering of its keys or a custom comparator.

Example: Using ArrayList

Below is an example of using `ArrayList` to store and retrieve elements:


import java.util.ArrayList;

public class Main {
    public static void main(String[] args) {
        // Create an ArrayList
        ArrayList fruits = new ArrayList<>();

        // Add elements
        fruits.add("Apple");
        fruits.add("Banana");
        fruits.add("Orange");

        // Get an element
        System.out.println(fruits.get(1)); // Output: Banana

        // Loop through the list
        for (String fruit : fruits) {
            System.out.println(fruit);
        }

        // Remove an element
        fruits.remove("Apple");
    }
}

Example: Using HashMap

The following example demonstrates how to use `HashMap` to store key-value pairs:


import java.util.HashMap;

public class Main {
    public static void main(String[] args) {
        // Create a HashMap
        HashMap ageMap = new HashMap<>();

        // Add key-value pairs
        ageMap.put("Alice", 30);
        ageMap.put("Bob", 25);
        ageMap.put("Charlie", 35);

        // Get a value by key
        System.out.println("Alice's age: " + ageMap.get("Alice"));

        // Iterate over keys and values
        for (String name : ageMap.keySet()) {
            System.out.println(name + " is " + ageMap.get(name) + " years old.");
        }
    }
}

Methods of the Collection Interface

Some common methods provided by the `Collection` interface:

  • add(E element): Adds the specified element to the collection.
  • remove(Object element): Removes the specified element from the collection, if it exists.
  • size(): Returns the number of elements in the collection.
  • clear(): Removes all elements from the collection.
  • contains(Object element): Returns `true` if the collection contains the specified element.
  • iterator(): Returns an iterator over the elements in the collection.
