Streams and Lambda Expressions

Understanding Streams

Streams are sequences of data elements supporting sequential and parallel aggregate operations. They can be created from various data sources, such as collections, arrays, or I/O channels.

Streams do not store data; they simply convey data from a source through a pipeline of computational operations.

Creating Streams

Example:
List<String> names = Arrays.asList("Alice", "Bob", "Charlie");
Stream<String> stream = names.stream();

Stream Operations

Stream operations are divided into intermediate and terminal operations:

  • Intermediate operations: return a new stream and can be chained (e.g., filter, map, sorted).
  • Terminal operations: produce a result or side-effect (e.g., forEach, collect, reduce).

Understanding Lambda Expressions

Lambda expressions are a way to implement functional interfaces (interfaces with a single abstract method) in a concise way. They provide a clear and expressive syntax for representing one method interfaces using an expression.

Example:
Runnable runnable = () -> System.out.println("Hello, World!");

Using Lambda Expressions with Streams

Lambda expressions are commonly used with streams to define the behavior of operations like filtering and mapping:

Example:
List<String> filteredNames = names.stream()
.filter(name -> name.startsWith("A"))
.collect(Collectors.toList());

Benefits of Streams and Lambda Expressions

  • Conciseness and clarity in code.
  • Ability to express complex data processing queries.
  • Support for parallel processing.
